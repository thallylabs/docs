---
title: Migrando a Thally
description: Mueve tu documentación existente de Mintlify, Docusaurus, GitBook o cualquier plataforma basada en Markdown a Thally con un solo comando.
---

El comando `create-thally-docs migrate` importa un sitio público de documentación o un repositorio de GitHub y lo convierte en un proyecto Thally en funcionamiento — sin copiar manualmente ni reconstruir la navegación.

```bash
npx create-thally-docs@latest migrate https://docs.example.com my-docs
```

---

## Cómo funciona

<Steps>

<Step title="Elegir la plataforma fuente">

En una migración interactiva, Thally pregunta si Mintlify, Docusaurus u otra plataforma aloja la documentación. Para scripts y CI, usa `--platform mintlify`, `--platform docusaurus` o `--platform auto`.

`--yes` omite todos los prompts. Si lo usas sin `--platform`, Thally utiliza la detección automática.

</Step>

<Step title="Descubrir páginas y navegación">

Para una URL pública, Thally lee el sitemap, las barras laterales renderizadas, las redirecciones y el contenido dentro de la ruta enviada. Para una URL de GitHub, hace un clon superficial y lee la configuración sin ejecutarla.

Las importaciones de repositorios reconocen estos archivos:

| Archivo | Plataforma detectada |
|------|-------------------|
| `docs.json` / `mint.json` | Mintlify |
| `docusaurus.config.js` + `sidebars.js` | Docusaurus |
| `book.yaml` | GitBook |
| `_meta.json` | Nextra |
| `vitepress.config.*` | VitePress |
| `.starlight/` | Starlight |
| *(ninguno de los anteriores)* | Se usa la estructura de directorios |

El directorio de documentación también se detecta automáticamente: `docs/`, `documentation/`, `content/`, `src/content/`, `pages/`, o la raíz del repositorio. También se descubren proyectos Docusaurus dentro de monorepos e instancias adicionales de `plugin-content-docs`.

</Step>

<Step title="Importar todos los archivos de documentación">

Las migraciones de repositorios leen cada archivo `.md` y `.mdx`. Las migraciones desde sitios públicos convierten la documentación renderizada a MDX portátil. Se conservan el frontmatter, los encabezados, enlaces, bloques de código, tablas, pestañas y avisos cuando la fuente los expone.

Los archivos que no son Markdown (p. ej. `.rst`) se convierten a MDX usando Claude si proporcionas una clave API de Anthropic.

</Step>

<Step title="Conectar la referencia de API">

Si se encuentra una especificación OpenAPI (`openapi.json`, `openapi.yaml`, `swagger.json`, etc.) en cualquier parte del repositorio, se copia a `public/` y la pestaña de Referencia de API en `docs.json` se configura automáticamente.

</Step>

<Step title="Escribir el proyecto Thally">

Todas las páginas convertidas se escriben en `src/content/`. La navegación de `docs.json` se construye a partir de la configuración de plataforma detectada o se infiere de la estructura de directorios.

</Step>

</Steps>

## Plataformas compatibles

### Elegir una plataforma

Ejecuta el comando sin `--yes` para elegir de forma interactiva:

```bash
npx create-thally-docs@latest migrate https://relay.dev/ relay-docs --max-pages 1000
```

Para una migración no interactiva, selecciona la plataforma explícitamente:

```bash
npx create-thally-docs@latest migrate https://relay.dev/ relay-docs \
  --platform docusaurus \
  --max-pages 1000 \
  --yes
```

<Warning>
`--yes` omite la pregunta de plataforma junto con todos los demás prompts. Úsalo con `--platform` cuando conozcas la plataforma; de lo contrario, la CLI usa detección automática.
</Warning>

### Mintlify

```bash
npx create-thally-docs migrate https://github.com/mintlify/starter my-docs
```

La navegación se lee desde `docs.json` (Mintlify v3) o `mint.json`. Las especificaciones OpenAPI en el directorio `api-reference/` se detectan y conectan automáticamente.

### Docusaurus

```bash
npx create-thally-docs@latest migrate https://relay.dev/ relay-docs --platform docusaurus
```

En sitios públicos, Thally reconstruye la navegación de la versión actual, sigue redirecciones canónicas, conserva rutas sensibles a mayúsculas y excluye árboles de versiones antiguas o `next`. Los avisos, pestañas, tablas y bloques de código resaltados se convierten a MDX compilable de Thally.

En repositorios, `sidebars.js` y sus equivalentes TypeScript se analizan de forma estática. Se admiten monorepos, categorías autogeneradas, assets, snippets `@site` y plugins de documentación adicionales. La configuración fuente nunca se ejecuta.

El importador se verificó con los conjuntos completos de documentación actual de [Relay](https://relay.dev/), [Algolia DocSearch](https://docsearch.algolia.com/docs/what-is-docsearch) y [Hasura DDN](https://hasura.io/docs/3.0/index/).

### GitBook

```bash
npx create-thally-docs migrate https://github.com/your-org/gitbook-docs my-docs
```

`SUMMARY.md` se usa para inferir el orden de navegación. Los bloques `{% hint %}` se convierten al componente de llamada de atención correspondiente de Thally.

### Markdown simple

Cualquier repositorio con archivos Markdown funciona — la navegación se construye a partir de la estructura de directorios.

```bash
npx create-thally-docs migrate https://github.com/your-org/docs my-docs
```

## Migrar a un proyecto existente

Usa `--into` para agregar contenido de otro repositorio a un proyecto Thally existente sin sobreescribirlo. Las nuevas pestañas y grupos se fusionan; el contenido existente no se modifica.

```bash
npx create-thally-docs migrate https://github.com/your-org/api-docs --into ./my-docs
```

---

## Opciones

| Indicador | Descripción |
|------|-------------|
| `--into <dir>` | Fusionar en un proyecto existente |
| `--branch <name>` | Anular la rama de git detectada automáticamente |
| `--docs-dir <path>` | Anular el subdirectorio de documentación detectado automáticamente |
| `--platform <name>` | Adaptador fuente: `mintlify`, `docusaurus` o `auto` |
| `--max-pages <count>` | Máximo de páginas de un sitio público que se importarán (`1`–`1000`) |
| `--api-key <key>` | Clave API de Anthropic (o establecer `ANTHROPIC_API_KEY`) para conversión de archivos no Markdown |
| `--yes`, `-y` | Aceptar valores predeterminados y omitir todos los prompts, incluida la selección de plataforma |

## A través del servidor MCP

Si usas Claude Code o Claude Desktop, puedes iniciar una migración a través del chat:

```
Migrate the Mintlify starter repo into a new Thally project at ~/dev/my-docs
```

La herramienta MCP `migrate_docs` acepta `platform` y `maxPages` para obtener el mismo comportamiento determinista. Consulta [Servidor MCP](/guides/mcp-server) para instrucciones de configuración.

## Después de la migración

Una vez completada la migración, inicia el servidor de desarrollo para revisar el resultado:

```bash
cd my-docs
npm run dev
```

Luego ejecuta `create-thally-docs check` para detectar cualquier problema antes de publicar:

```bash
npx create-thally-docs check my-docs
```

Tareas comunes posteriores a la migración:
- Actualiza `src/data/site.ts` con tu logotipo y colores de marca
- Reemplaza las imágenes de marcador de posición en `src/content/`
- Revisa el orden de navegación en `docs.json`
- Establece `THALLY_SITE_URL` antes de desplegar
